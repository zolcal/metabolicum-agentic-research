# Open questions

Most setup decisions that were previously blockers have been resolved. Standard Medical anchor inputs now exist through the SM range YAML waves and database-backed marker work; research target envelope inputs are system-generated from `research_target_envelopes` rather than hand-maintained per-marker YAMLs; provider account ownership and endpoint routing are captured in `REVIEW-2026-05-21-llm-access.md`. The remaining launch blocker is project provisioning: create and initialize the standalone `metabolicum-agentic-research` repository, deploy its Supabase schema, create the local runtime filesystem, and enforce the secrets boundary from both `metasync` and the legacy `metabolicum-research` workspace. Translation quota policy — pre-run quota check, dry-run estimate, hard-stop threshold, and local or manual fallback before non-English ingestion — is an orchestration-runtime concern, not a contract blocker.

The evidence-grade sub-grade semantics are resolved in section fifteen. `A` through `E` parent tiers represent evidence families, numeric suffixes rank strength within the parent tier, `D3` means human mechanistic or physiological evidence, and non-human preclinical evidence uses `P1` for animal studies and `P2` for in vitro, cell-line, organoid, or non-human mechanistic models.

The practitioner/source registry format, default source-tier weights, commercial-interest schema, and first implementation seed sizing are resolved in section sixteen. The remaining operational work is to populate the runtime registry from the section-sixteen seed roster and add marker-specific sources when discovery gaps appear.

Podcast augmentation and fallback procurement should be chosen after the verified RSS feed list is populated. RSS feeds are primary; Podscan, Listen Notes, and AssemblyAI only cover discovery and transcription gaps, so the subscription decision should be based on the number of gaps remaining after the RSS pass.

Two governance questions remain. The legal review boundary: who adjudicates the ambiguous cases where the legal agent returns `approve_with_modification`? `[JUDGMENT]` Default: the user adjudicates until a delegated reviewer is explicitly named. The production migration approval process: who signs off on each manual review of the migration query before execution? `[JUDGMENT]` Default: the user signs off on every `metasync` import until a written delegation rule exists.

Several strategic decisions can wait but should be acknowledged. Full multilingual discovery is deferred beyond the pilot contract; the first implementation records source language, preserves original verbatim quotes, and records translation method, but does not require all languages to be fully supported at launch. Semantic embedding model choice, local GPU fallback models, existing-corpus calibration, and network-derived authority scoring are deferred runtime or later calibration decisions. Prior semantic-search work on `multilingual-e5-large` and an approximate 0.76 threshold should be preserved as calibration evidence, not treated as a hard-coded mandate. The exact algorithm for widening target envelopes across all available good range data remains a data-generation decision, not a documentation blocker. The public-versus-private surface for these documents is a question — the recommendation is private until sensitive implementation details are removed, then consider opening up as a Metabolicum credibility signal. The marker re-sweep cadence remains an implementation policy question.

What is explicitly out of scope for the first implementation includes Facebook and Instagram (per section three), LinkedIn automated scraping (per section three), the PM paradigm (referenced in Metabolicum's broader platform philosophy but not produced by this pipeline), real-time alerting (the pipeline is batch), user-facing UI changes beyond the internal dashboard (backend only), and multi-tenant pipelines (one user, one Metabolicum).
